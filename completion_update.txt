Mini Redis Completion Update

추가 목적
- 재시작 복구를 실제 서버 경로에 연결했다.
- router 를 문서 수준이 아니라 dispatcher 판단 경로에 들어가게 만들었다.
- 이제 persistence / cluster 는 다음 단계가 아니라, 최소 완성 형태로 실행 경로에 포함된다.

이번 단계 핵심 변경
1. restart recovery 연결
- 서버 시작 시 data_dir 아래 appendonly.aof 가 있으면 우선 replay 한다.
- aof 가 없으면 dump.rdb snapshot 을 로드한다.
- SAVE 명령을 dispatcher 에 추가해서 현재 메모리 상태를 dump.rdb 로 저장할 수 있다.
- write 명령 SET / DEL / EXPIRE 는 성공 후 appendonly.aof 에 기록된다.

2. storage snapshot/load 지원
- storage 가 현재 메모리 상태와 expiration metadata 를 export 할 수 있다.
- restart 시 snapshot/aof 결과를 storage 로 다시 load 할 수 있다.
- expired key 는 load 과정에서 걸러진다.

3. cluster routing 구조 통합
- ClusterCoordinator 를 추가했다.
- dispatcher 는 key 기반 명령 전에 현재 노드가 해당 key owner 인지 확인할 수 있다.
- 현재 기본 실행은 단일 노드라 항상 local 처리된다.
- 다중 노드 구성이 들어오면 remote owner 에 대해 MOVED 에러를 반환할 수 있다.

4. 실행 경로 변화
- main.build_server() 가 storage + persistence + cluster + dispatcher + protocol + tcp server 를 조립한다.
- python -m src.main --data-dir <dir> 로 persistence 파일 경로를 지정할 수 있다.
